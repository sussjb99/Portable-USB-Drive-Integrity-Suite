# Contributing to PUDIS (Portable-USB-Drive-Integrity-Suite)

Thank you for your interest in contributing to Portable Drive Baby Sitter.  
This project focuses on reliability, data integrity, and long‑term maintainability.  
Contributions of all kinds are welcome — code, documentation, testing, and issue reports.

---

## 🧭 How to Contribute

### 1. Open an Issue First
Before submitting a pull request, please open an issue describing:

- The problem you’re solving  
- The proposed change or feature  
- Any relevant logs, screenshots, or test results  

This helps ensure the work aligns with the project’s goals and avoids duplicate effort.

---

## 🛠 Code Contributions

### Coding Standards
- Keep code readable, modular, and well‑commented.  
- Avoid unnecessary dependencies.  
- Follow the existing style of the project (C++, PowerShell, and batch scripting).  
- Use clear, descriptive commit messages.

### Pull Requests
When submitting a PR:

1. Fork the repository  
2. Create a feature branch  
3. Make your changes  
4. Ensure the project builds and runs cleanly  
5. Submit a pull request referencing the related issue  

PRs should include:

- A summary of the change  
- Why the change is needed  
- Any testing performed  
- Any known limitations  

---

## 🧪 Testing

Before submitting a PR, please test your changes on:

- At least one USB flash drive  
- At least one HDD/SSD (if available)  
- Both small and large datasets when applicable  

If your change affects integrity scanning or surface analysis, include test notes in the PR.

---

## 📄 Documentation

Documentation improvements are always welcome.  
This includes:

- README updates  
- Clarifying instructions  
- Adding examples  
- Improving error messages  
- Expanding FAQs  

---

## 🔐 Security & Safety

If you discover a bug that could cause data loss, corruption, or security issues:

**Do not open a public issue.**  
Instead, contact the maintainer privately through GitHub’s private issue feature.
